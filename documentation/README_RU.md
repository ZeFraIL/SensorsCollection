## 🎯 Цель проекта
•	Приложение предназначено для демонстрации работы с различными аппаратными датчиками (сенсорами) Android-устройства.  
•	Это важно для понимания взаимодействия программного обеспечения с аппаратной частью смартфона.  
•	Целевая аудитория: разработчики, студенты и пользователи, интересующиеся возможностями своих устройств.
________________________________________
## 📌 Требования к приложению
**Функциональные требования**
•	Отображение списка всех доступных сенсоров на устройстве.  
•	Получение и визуализация данных в реальном времени от гироскопа, магнитометра и датчика ориентации.  
•	Подсчет шагов (шагомер).  
•	Измерение уровня освещенности с изменением цвета интерфейса.  
•	Распознавание встряхивания устройства (Shake).

**Нефункциональные требования**
•	Производительность: обновление данных без задержек.  
•	Удобство использования: интуитивно понятное меню с кнопками.  
•	Надёжность: корректная обработка жизненного цикла Activity (регистрация/отписка от датчиков).

## 🧠 Общая архитектура
•	**Выбранный подход:**  
–	MVC (Model-View-Controller) — классический подход для Android на Java.

•	**Почему выбран именно он:**  
–	Простота реализации для образовательного проекта. Прямое взаимодействие Activity с SensorManager позволяет наглядно показать принципы работы API.

•	**Основные компоненты системы:**  
–	MainActivity: навигационный контроллер.  
–	SensorActivities: специализированные экраны для каждого типа датчика.  
–	SensorManager: системный сервис для доступа к датчикам.

## 🧩 Подробное описание классов
📌 **Класс: MainActivity**  
**Роль:** Точка входа и главное меню.  
**Ответственность:** Динамическое создание интерфейса (кнопок) и запуск соответствующих Activity.  
**Основные методы:**  
- `onCreate()` — инициализация экрана.  
- `initComponents()` — программное создание кнопок на основе массива классов и названий.  
**Взаимодействие:** Запускает все остальные Activity через Intent.

📌 **Класс: LightSensor**  
**Роль:** Обработка данных датчика освещенности.  
**Почему используется:** Для демонстрации изменения UI в зависимости от внешней среды.  
**Основные методы:**  
- `onSensorChanged()` — логика изменения цвета фона (от черного до белого).

📌 **Класс: ShakeAndStart**  
**Роль:** Детектор встряхивания.  
**Ответственность:** Отслеживание резких ускорений по осям X, Y, Z.

## 🔄 Диаграмма работы приложения
1. Пользователь запускает приложение.
2. `MainActivity` создает список кнопок.
3. Пользователь выбирает датчик (например, "LightSensor").
4. Открывается новая Activity, регистрируется слушатель `SensorEventListener`.
5. При изменении физических параметров датчик шлет событие, UI обновляется.
6. При закрытии экрана слушатель отключается (`onPause`).

## 🎨 UI/UX анализ
•	Интерфейс построен на динамическом создании элементов в `MainActivity`, что делает его гибким.  
•	Использованы принципы:
–	**Простота:** Минималистичный дизайн, фокус на данных.
–	**Логичность:** Одна кнопка — одна функция.
–	**Доступность:** Крупный текст и контрастные цвета (особенно в LightSensor).

## ⚙️ Работа с потоками
•	Используются: Системные колбэки `onSensorChanged`.  
•	**Почему выбран этот способ:** Это стандартный механизм Android SDK для датчиков.  
•	**Предотвращение зависаний:** Легкая обработка данных в методах обратного вызова.  
•	**Утечки памяти:** Использование `unregisterListener` в `onPause` гарантирует освобождение ресурсов.
________________________________________
## 💾 Работа с данными

•	**Корректность:** Обеспечивается прямой трансляцией значений с аппаратных датчиков.
