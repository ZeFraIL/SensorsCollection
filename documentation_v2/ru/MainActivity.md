# Подробное описание класса: MainActivity

## 1. Общая информация
*   **Имя класса:** `MainActivity`
*   **Тип:** `Activity` (Экран)
*   **Назначение:** Этот класс является главной точкой входа в приложение и его домашним экраном. Он отвечает за отображение меню из кнопок, которые позволяют пользователю переходить к различным экранам демонстрации датчиков.
*   **Взаимодействие:** Он взаимодействует со всеми остальными Activity (`ListAllSensors`, `GyroscopeSensor`, `MagnetField`, `OrientationSensor`, `StepByStep`, `LightSensor`, `ShakeAndStart`), запуская их с помощью объектов `Intent` при нажатии на соответствующую кнопку.

## 2. Переменные (Поля класса)

| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | Хранит состояние/окружение приложения. | В `initComponents()` для создания кнопок и `Intent`. |
| `names` | `String[]` | Массив текстовых меток для кнопок меню. | В `initComponents()` для установки текста кнопок. |
| `n` | `int` | Хранит количество создаваемых кнопок. | В `initComponents()` как ограничение цикла. |
| `buttons` | `Button[]` | Массив для хранения динамически созданных объектов `Button`. | В `initComponents()` для хранения и настройки интерфейса. |
| `LLallButtons`| `LinearLayout` | Контейнер разметки, в который добавляются кнопки. | В `initComponents()` для добавления View. |
| `classes` | `Class[]` | Массив имен классов (Activity), соответствующих кнопкам. | В `initComponents()` для определения цели `Intent`. |

## 3. Методы класса

### Имя метода: `onCreate`
*   **Тип метода:** `protected`
*   **Возвращаемое значение:** `void` (ничего не возвращает)
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `savedInstanceState` | `Bundle` | Хранит состояние активности, если она была ранее закрыта. |
*   **Что делает метод:**
    1. Вызывает метод `onCreate` родительского класса.
    2. Устанавливает визуальную разметку из файла `activity_main.xml`.
    3. Вызывает `initComponents()` для построения меню.
*   **Когда вызывается:** Автоматически системой Android при запуске приложения.
*   **Важно:** Это первый этап жизни Activity.

### Имя метода: `initComponents`
*   **Тип метода:** `private`
*   **Возвращаемое значение:** `void`
*   **Параметры:** Нет
*   **Что делает метод:**
    1. Инициализирует `context` текущим экземпляром активности.
    2. Проходит циклом по массиву `names`.
    3. Для каждого имени программно создает новую кнопку `Button`.
    4. Настраивает свойства кнопки (отступы, размеры текста).
    5. Создает `Intent`, указывающий на соответствующий класс из массива `classes`.
    6. Прикрепляет `ClickListener` к кнопке, чтобы запускать выбранную Activity при нажатии.
    7. Добавляет кнопку в контейнер `LLallButtons`.
*   **Когда вызывается:** Вручную из `onCreate()`.
*   **Важно:** Такой подход позволяет очень легко расширять меню: достаточно добавить название и класс в массивы.

## 4. Жизненный цикл
*   **`onCreate()`**: Вызывается при первом создании Activity. Настраивает разметку и инициализирует кнопки.

## 5. Взаимодействие с интерфейсом (UI)
*   **Элементы:** `LinearLayout` (с ID `LLallButtons`) и динамически создаваемые `Button`.
*   **Связь с кодом:** `findViewById` используется для получения макета. Кнопки создаются через `new Button(context)`.
*   **События:** `setOnClickListener` обрабатывается для каждой кнопки для перехода на другие экраны.

## 6. Взаимодействие с другими компонентами
*   **Intents:** Использует `Intent` для перехода от `MainActivity` к конкретным активностям датчиков.
*   **Передача данных:** Сложные данные не передаются, только команда на запуск нового экрана.

## 7. Общая логика
1. Приложение запускается и отображает основной макет.
2. Код считывает массивы названий и типов классов.
3. Он «рисует» кнопки на экране для каждого датчика.
4. При нажатии на кнопку приложение открывает экран конкретного датчика.

## 8. Упрощенное объяснение
**«Объяснение простыми словами»**
Представьте, что `MainActivity` — это **стойка регистрации** или **главное меню** игры.
Представьте, что вы находитесь в здании со множеством комнат (датчиков). `MainActivity` — это коридор с дверями. У каждой двери есть табличка (Кнопка). Когда вы нажимаете на табличку, вы фактически проходите через эту дверь в комнату, где отображаются данные конкретного датчика.

---
**Заметка для студентов:** В `initComponents` кнопки создаются «динамически» (через код), а не «статически» (в XML). Это профессиональный способ обработки списков, размер которых может меняться, хотя для фиксированного меню это также можно было бы сделать в XML для лучшей производительности.
