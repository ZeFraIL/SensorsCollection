# Подробное описание класса: GyroscopeSensor

## 1. Общая информация
*   **Имя класса:** `GyroscopeSensor`
*   **Тип:** `Activity` (Экран)
*   **Назначение:** Эта Activity измеряет скорость вращения устройства вокруг трех его физических осей (X, Y и Z). Она используется для определения того, как быстро телефон поворачивается или наклоняется.
*   **Взаимодействие:** Запускается из `MainActivity`. Использует систему ручной регистрации слушателя датчика.

## 2. Переменные (Поля класса)

| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `tvGX, tvGY, tvGZ`| `TextView` | Отображает значения скорости вращения для осей X, Y и Z. | В `onSensorChanged()`. |
| `bRegL` | `Button` | Кнопка «Register». Запускает датчик. | В `onCreate()`. |
| `bUnRegL` | `Button` | Кнопка «Unregister». Останавливает датчик. | В `onCreate()`. |
| `sensorManager`| `SensorManager` | Системный сервис для доступа к датчикам. | Во всем классе. |
| `gyroscopeSensor`| `Sensor` | Ссылка на оборудование гироскопа. | Для запуска/остановки прослушивания. |
| `gyroscopeListener`| `SensorEventListener`| Логика, обрабатывающая входящие данные. | Определена в `onCreate()`. |

## 3. Методы класса

### Имя метода: `onCreate`
*   **Что делает метод:**
    1. Настраивает пользовательский интерфейс.
    2. Вызывает `initComponents()`.
    3. Находит датчик гироскопа. Если он отсутствует, выводит предупреждение.
    4. Определяет `gyroscopeListener`. Когда датчик фиксирует движение, он берет значения скоростей X, Y, Z и обновляет текст на экране.
    5. Настраивает кнопку `bRegL` на запуск прослушивания при нажатии.
    6. Настраивает кнопку `bUnRegL` на остановку прослушивания и сброс текста на «?» при нажатии.
*   **Когда вызывается:** При создании Activity.

### Имя метода: `onSensorChanged` (Внутри слушателя)
*   **Параметры:** `SensorEvent event`.
*   **Что делает метод:**
    1. Проверяет, поступили ли данные от гироскопа.
    2. Считывает `values[0]` (X), `values[1]` (Y) и `values[2]` (Z).
    3. Обновляет текстовые метки на экране этими числами.
*   **Когда вызывается:** Каждый раз, когда устройство вращается.

## 4. Жизненный цикл
*   **`onCreate()`**: Основная настройка.
*   **Особенность:** Этот класс **не** отключает датчик автоматически при закрытии экрана (отсутствует отписка в `onPause`). Пользователь должен нажать кнопку «Unregister» вручную.

## 5. Взаимодействие с интерфейсом (UI)
*   **Элементы:** 3 текстовых поля (`TextView`) и 2 кнопки (`Button`).
*   **Взаимодействие:** Пользователь имеет полный контроль над тем, когда датчик активен.

## 6. Взаимодействие с другими компонентами
*   **Оборудование:** Прямое взаимодействие с гироскопом через `SensorManager`.

## 7. Общая логика
1. Пользователь открывает экран гироскопа.
2. Он видит значения скорости в виде «?».
3. Он нажимает «Register».
4. Телефон начинает измерять вращение. Когда пользователь двигает телефон, числа на экране быстро меняются.
5. Он нажимает «Unregister», чтобы остановиться.

## 8. Упрощенное объяснение
**«Объяснение простыми словами»**
Представьте, что внутри вашего телефона есть **спиннер**. Когда вы поворачиваете телефон, спиннер реагирует. Класс `GyroscopeSensor` — это как **спидометр** для этого спиннера. Он говорит вам точно, как быстро вы вращаете телефон в трех разных направлениях.

---
**Заметка для студентов:** В реальном приложении всегда следует отключать слушателей в `onPause()`, чтобы датчик не продолжал работать в фоновом режиме, если пользователь забыл нажать «Unregister».
