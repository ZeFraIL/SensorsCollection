# Подробное описание класса: ShakeAndStart

## 1. Общая информация
*   **Имя класса:** `ShakeAndStart`
*   **Тип:** `Activity` (Экран)
*   **Назначение:** Эта Activity определяет, встряхивает ли пользователь устройство. Как только фиксируется значительное встряхивание, запускается голосовой обратный отсчет, и пользователь автоматически возвращается на главный экран (`MainActivity`).
*   **Взаимодействие:** Запускается из `MainActivity` и возвращается в нее после завершения отсчета. Использует датчик акселерометра и систему синтеза речи (Text-To-Speech).

## 2. Переменные (Поля класса)

| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `SHAKE_LEVEL` | `int` | Порог чувствительности (установлен на 14). Чем выше значение, тем сильнее нужно трясти. | В `onSensorChanged()` для проверки встряхивания. |
| `tts` | `TextToSpeech` | Движок для преобразования текста в голос. | В `onSensorChanged()` и `onTick()`. |
| `stadb` | `String[]` | Массив фраз для голосового отсчета. | В `onTick()` для произношения. |
| `sensor_shake` | `Sensor` | Ссылка на датчик акселерометра. | В `initComponents()` и `onCreate()`. |
| `gotostart` | `Intent` | Команда для возврата в `MainActivity`. | В `onFinish()` таймера. |

## 3. Методы класса

### Имя метода: `initComponents`
*   **Тип метода:** `private`
*   **Возвращаемое значение:** `void`
*   **Что делает метод:**
    1. Инициализирует `Intent` для навигации.
    2. Получает `SensorManager` и датчик `ACCELEROMETER`.
    3. Определяет `sensorEventListener`, который вычисляет «сумму встряхивания»: `|X| + |Y| + |Z|`.
    4. Если сумма превышает `SHAKE_LEVEL`:
        - Инициализирует движок `TextToSpeech`.
        - Запускает таймер `CountDownTimer` на 8 секунд.
*   **Когда вызывается:** Из `onCreate()`.

### Имя метода: `onTick` (внутри CountDownTimer)
*   **Что делает:** Каждые 2 секунды выбирает следующую фразу из массива `stadb` и произносит ее вслух с помощью `tts.speak()`.
*   **Когда вызывается:** Автоматически таймером.

### Имя метода: `onFinish` (внутри CountDownTimer)
*   **Что делает:**
    1. Отключает слушателя датчика, чтобы прекратить обнаружение встряхиваний.
    2. Запускает `MainActivity`.
*   **Когда вызывается:** По окончании 8-секундного отсчета.

## 4. Жизненный цикл
*   **`onCreate()`**: Регистрирует слушателя датчика немедленно.
*   **Пропущенные методы:** В этом классе отсутствует отключение датчика в `onPause()` или `onStop()`.

## 5. Взаимодействие с интерфейсом (UI)
*   **Элементы:** Специфические кнопки не используются; все взаимодействие основано на физическом движении (тряске) и звуке.

## 6. Общая логика
1. Пользователь открывает экран.
2. Телефон ждет «встряски».
3. При встряхивании телефон говорит: «Три секунды до старта», «Две секунды...» и так далее.
4. После отсчета приложение «прыгает» обратно в главное меню.

## 7. Внимание, ошибки! (Важно для студентов) ⚠️
В этом коде есть две серьезные проблемы, о которых стоит упомянуть на защите:
1.  **Инициализация TTS:** Команда `new TextToSpeech(...)` вызывается внутри `onSensorChanged`. Это **очень плохо**. Если вы трясете телефон в течение 1 секунды, `onSensorChanged` может сработать 50 раз. Это создаст 50 объектов TTS, что, скорее всего, приведет к зависанию или вылету приложения.
    - *Исправление:* Инициализируйте `tts` один раз в `onCreate`.
2.  **Разряд батареи:** Датчик регистрируется в `onCreate`, но никогда не отключается, если пользователь просто выйдет из Activity (например, кнопкой «Назад») до завершения таймера.
    - *Исправление:* Всегда отключайте слушателей в `onPause()`.

## 8. Упрощенное объяснение
**«Объяснение простыми словами»**
Представьте, что ваш телефон — это **спящий охранник**. Когда вы его трясете, вы его «будите». Охранник начинает считать вслух: «3, 2, 1... ПОЕХАЛИ!». Как только он заканчивает счет, он отправляет вас обратно к главному входу в здание.

---
**Заметка для студентов:** Использование `Math.abs(x) + Math.abs(y) + Math.abs(z)` — это простой способ обнаружить движение в любом направлении!
