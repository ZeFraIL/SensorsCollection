# Подробное описание класса: OrientationSensor

## 1. Общая информация
*   **Имя класса:** `OrientationSensor`
*   **Тип:** `Activity` (Экран)
*   **Назначение:** Эта Activity отслеживает ориентацию (наклон) устройства с помощью датчика акселерометра. Она показывает, как телефон расположен в пространстве по осям X, Y и Z.
*   **Взаимодействие:** Запускается из `MainActivity`. Регистрирует датчик сразу после создания экрана.

## 2. Переменные (Поля класса)

| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `sensorManager`| `SensorManager` | Системный сервис для доступа к оборудованию устройства. | В `onCreate()` и `onStop()`. |
| `orientationSensor`| `Sensor` | Ссылка на датчик акселерометра. | В `onCreate()`. |
| `orientationListener`| `SensorEventListener`| Получает и обрабатывает данные об ускорении. | Определено в `onCreate()`. |
| `tvX, tvY, tvZ`| `TextView` | Отображает необработанные значения ускорения. | В `onSensorChanged()`. |

## 3. Методы класса

### Имя метода: `onCreate`
*   **Что делает метод:**
    1. Подключает компоненты пользовательского интерфейса.
    2. Получает `SensorManager`.
    3. Находит датчик `TYPE_ACCELEROMETER`.
    4. Создает слушателя, который обновляет `TextView` значениями гравитации/ускорения (X, Y, Z).
    5. Начинает прослушивание датчика.
*   **Когда вызывается:** При создании Activity.

### Имя метода: `onStop`
*   **Что делает метод:** Останавливает слушателя датчика для экономии заряда батареи, когда пользователь покидает экран.

## 4. Жизненный цикл
*   **`onCreate()`**: Этап запуска.
*   **`onStop()`**: Этап очистки.

## 5. Взаимодействие с интерфейсом (UI)
*   **Элементы:** 3 объекта `TextView` для осей X, Y и Z.

## 6. Общая логика
1. Пользователь наклоняет телефон.
2. Акселерометр фиксирует изменение силы тяжести на каждой оси.
3. Приложение отображает эти числа. Например, если телефон лежит плашмя на столе, ось Z должна показывать примерно 9,8 (земное притяжение).

## 7. Внимание, ошибки! (Важно для студентов) ⚠️
- **Путаница в названиях:** В коде датчик назван `orientationSensor`, но на самом деле он запрашивает `Sensor.TYPE_ACCELEROMETER`. Хотя акселерометры помогают вычислить ориентацию, это не одно и то же.
- **Лучший подход:** Для точной ориентации (компас/уровень) разработчикам следует использовать `Sensor.TYPE_ROTATION_VECTOR` или объединять данные акселерометра и магнитометра.

## 8. Упрощенное объяснение
**«Объяснение простыми словами»**
Представьте, что внутри вашего телефона находится **чаша с водой**. Когда вы наклоняете телефон, вода движется. Этот класс считывает, куда именно плещется вода. Он сообщает телефону, лежит ли он плашмя, стоит или наклонен в сторону.

---
**Заметка для студентов:** Необработанные данные акселерометра включают силу тяжести (9,8 м/с²). Если вы уроните телефон, все значения на мгновение обнулятся!
