# Подробное описание класса: StepByStep

## 1. Общая информация
*   **Имя класса:** `StepByStep`
*   **Тип:** `Activity` (Экран)
*   **Назначение:** Эта Activity работает как шагомер. Она использует датчик счетчика шагов устройства, чтобы отслеживать, сколько шагов сделал пользователь, пока приложение активно.
*   **Взаимодействие:** Запускается из `MainActivity`. Запрашивает у пользователя специальное разрешение (`ACTIVITY_RECOGNITION`) для доступа к данным о физической активности.

## 2. Переменные (Поля класса)

| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `all_steps` | `int` | Хранит общее количество шагов, подсчитанных в текущем сеансе. | В `onSensorChanged()` для обновления счетчика. |
| `tv_sbs` | `TextView` | Отображает текст со счетчиком шагов на экране. | В `onSensorChanged()` и `initComponents()`. |
| `iv_sbs` | `ImageView` | Отображает изображение идущего или стоящего человека. | В обработчиках нажатий `bStart` и `bStop`. |
| `stepCounterSensor` | `Sensor` | Ссылка на физический датчик счетчика шагов. | В `initComponents()` и `registerListener()`. |

## 3. Методы класса

### Имя метода: `onCreate`
*   **Что делает метод:**
    1. Запрашивает разрешение `ACTIVITY_RECOGNITION` (требуется для Android 10+).
    2. Инициализирует компоненты.
    3. Настраивает кнопку **Start**: показывает изображение «идущего человека» и начинает прослушивание датчика.
    4. Настраивает кнопку **Stop**: показывает изображение «стоящего человека» и прекращает прослушивание.
*   **Когда вызывается:** При создании Activity.

### Имя метода: `onSensorChanged`
*   **Что делает метод:**
    1. Вызывает небольшое сообщение «Toast» с текстом «Step».
    2. Получает количество шагов из `event.values[0]`.
    3. Прибавляет значение к `all_steps` и обновляет `TextView`.
*   **Когда вызывается:** Каждый раз, когда датчик фиксирует шаг.

## 4. Жизненный цикл
*   **`onCreate()`**: Настройка и запрос разрешения.
*   **Важно:** Этот класс полагается на ручной запуск/остановку через кнопки, а не на методы `onResume`/`onPause`.

## 5. Взаимодействие с интерфейсом (UI)
*   **Элементы:** `ImageView` (`iv_sbs`), `TextView` (`tv_sbs`), `Button` (`bStart`, `bStop`).
*   **События:** Нажатия кнопок для управления датчиком.

## 6. Взаимодействие с другими компонентами
*   **Разрешения:** Использует `ActivityCompat.requestPermissions`. Это взаимодействие на уровне системы.

## 7. Внимание, ошибки! (Важно для студентов) ⚠️
В логике расчета шагов есть ошибка:
- **Ошибка:** `TYPE_STEP_COUNTER` возвращает общее количество шагов с момента **последней перезагрузки телефона**. Если датчик говорит «5000», и вы прибавляете это к `all_steps`, счетчик будет неверным.
- **Исправление:** Вы должны записать «начальное значение», когда пользователь нажимает «Start», а затем вычитать это начальное значение из текущего показания датчика, чтобы получить «шаги за этот сеанс».
- **Обработка разрешений:** Код запрашивает разрешение, но не проверяет, ответил ли пользователь «Да» или «Нет». Если они откажут, приложение может работать некорректно.

## 8. Упрощенное объяснение
**«Объяснение простыми словами»**
Представьте, что в вашем телефоне есть **шагомер**, похожий на те, что люди цепляют на пояс. Этот класс — **дисплей** для этого шагомера. Вы нажимаете «Start», чтобы увидеть идущего человечка, и пока вы идете, «мозг» приложения слушает «щелчки» от датчика и обновляет число на экране.

---
**Заметка для студентов:** `ACTIVITY_RECOGNITION` — это «опасное» разрешение, поэтому приложение должно явно запрашивать его у пользователя во время работы.
