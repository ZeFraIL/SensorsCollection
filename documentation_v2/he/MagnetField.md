# תיאור מפורט של המחלקה: MagnetField

## 1. מידע כללי
*   **שם המחלקה:** `MagnetField`
*   **סוג:** `Activity` (מסך)
*   **תפקיד:** Activity זו מודדת את השדה המגנטי סביב המכשיר. ניתן להשתמש בה כדי לזהות את הקטבים המגנטיים של כדור הארץ (כמו מצפן) או למצוא מגנטים וחפצי מתכת בקרבת מקום.
*   **אינטראקציה:** מופעלת מתוך `MainActivity`. היא מתחילה למדוד באופן אוטומטי בעת הפתיחה ומפסיקה כאשר המשתמש עוזב את המסך.

## 2. משתנים (שדות המחלקה)

| שם | סוג | מטרה | היכן בשימוש |
| :--- | :--- | :--- | :--- |
| `sensorManager`| `SensorManager` | שירות מערכת לניהול חיישנים. | ב-`onCreate()` וב-`onStop()`. |
| `magneticFieldSensor`| `Sensor` | הפניה לחומרת המגנטומטר. | ב-`onCreate()`. |
| `magneticFieldListener`| `SensorEventListener`| מטפל בנתונים מגנטיים נכנסים. | מוגדר ב-`onCreate()`. |
| `tvMX, tvMY, tvMZ`| `TextView` | מציגים את עוצמת השדה המגנטי עבור הצירים X, Y, Z. | ב-`onSensorChanged()`. |

## 3. מתודות המחלקה

### שם המתודה: `onCreate`
*   **מה המתודה עושה:**
    1. מגדירה את הפריסה.
    2. מאתחלת את ה-`TextView`-ים.
    3. מוצאת את חיישן ה-`MAGNETIC_FIELD`.
    4. מגדירה את המאזין: קורא את `values[0, 1, 2]` (נמדד ב-**micro-Tesla, µT**) ומציג אותם על המסך.
    5. רושמת את המאזין באופן מיידי כך שהמדידה מתחילה מיד.
*   **מתי נקראת:** כאשר ה-Activity מתחילה.

### שם המתודה: `onStop`
*   **מה המתודה עושה:** מבטלת את הרישום של המאזין המגנטי.
*   **מתי נקראת:** כאשר ה-Activity כבר אינה גלויה למשתמש.
*   **חשוב:** שימוש ב-`onStop` במקום ב-`onPause` הוא מקובל, אך `onPause` בדרך כלל עדיף לשחרור משאבים מהיר יותר.

## 4. מחזור חיים
*   **`onCreate()`**: אתחול והפעלת החיישן.
*   **`onStop()`**: הפסקת החיישן כדי לחסוך באנרגיה.

## 5. אינטראקציה עם הממשק (UI)
*   **אלמנטים:** 3 `TextView`-ים.
*   **אירועים:** ללא (עדכונים אוטומטיים).

## 6. לוגיקה כללית
1. המשתמש פותח את מסך המגנטומטר.
2. הטלפון מזהה את המשיכה המגנטית של הסביבה.
3. ערכי ה-X, Y וה-Z מתעדכנים ככל שהמשתמש מקרב את הטלפון לחתיכת מתכת או מסובב אותו לעבר הקוטב הצפוני.
