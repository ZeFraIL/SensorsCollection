# תיאור מפורט של המחלקה: LightSensor

## 1. מידע כללי
*   **שם המחלקה:** `LightSensor`
*   **סוג:** `Activity` (מסך)
*   **תפקיד:** Activity זו מודדת את רמת תאורת הסביבה באמצעות חיישן האור של המכשיר. היא מציגה תיאור טקסטואלי של עוצמת האור ומשנה את צבע הרקע של המסך כדי לשקף עד כמה הסביבה בהירה או חשוכה.
*   **אינטראקציה:** היא מופעלת מתוך `MainActivity`. היא מממשת את הממשק `SensorEventListener` כדי לקבל נתונים ישירות מחיישני החומרה של המכשיר.

## 2. משתנים (שדות המחלקה)

| שם | סוג | מטרה | היכן בשימוש |
| :--- | :--- | :--- | :--- |
| `LL` | `LinearLayout` | המיכל הראשי של הפריסה. משמש לשינוי צבע הרקע של המסך. | ב-`onCreate()` (אתחול) וב-`brightness()` (עדכון צבע). |
| `brightness` | `Sensor` | הפניה לחיישן האור הפיזי של המכשיר. | ב-`setUpSensor()` וב-`onResume()`. |
| `sensorManager`| `SensorManager` | שירות המערכת המנהל את כל החיישנים במכשיר. | ב-`setUpSensor()`, `onResume()`, ו-`onPause()`. |

## 3. מתודות המחלקה

### שם המתודה: `onCreate`
*   **מה המתודה עושה:**
    1. מגדירה את הפריסה מקובץ `activity_light_sensor.xml`.
    2. מחברת את המשתנה `LL` לאלמנט הפריסה באמצעות ה-ID שלו.
    3. קוראת ל-`setUpSensor()` כדי לאתחל את האינטראקציה עם החומרה.
*   **מתי נקראת:** כאשר המסך נפתח לראשונה.

### שם המתודה: `setUpSensor`
*   **מה המתודה עושה:**
    1. מקבלת את שירות ה-`SensorManager` מהמערכת.
    2. מבקשת מהמנהל את חיישן ה-"Light" (אור) המוגדר כברירת מחדל.
*   **מתי נקראת:** במהלך שלב ה-`onCreate()`.

### שם המתודה: `onSensorChanged`
*   **פרמטרים:** `SensorEvent sensorEvent` (מכיל נתונים על השינוי בחיישן).
*   **מה המתודה עושה:**
    1. בודקת אם הנתונים הנכנסים אכן מגיעים מחיישן האור.
    2. מחלצת את הערך הראשון (`values[0]`), המייצג את עוצמת האור ב-**lux**.
    3. קוראת למתודה `brightness()` כדי לקבל תיאור טקסט ולעדכן את צבע הממשק.
    4. מציגה הודעת "Toast" (חלון קופץ קטן) עם הנתונים.
*   **מתי נקראת:** באופן אוטומטי על ידי המערכת בכל פעם שרמת האור סביב המכשיר משתנה.

### שם המתודה: `brightness`
*   **ערך חזרה:** `String` (תיאור של רמת האור).
*   **פרמטרים:** `float light1` (ערך האור ב-lux).
*   **מה המתודה עושה:**
    - משתמשת במשפטי `if` כדי לסווג את ערך האור:
        - 0 lux: רקע שחור, מחזירה "Pitch black".
        - 1-10 lux: אפור כהה, מחזירה "Dark".
        - ... וכן הלאה עד "Incredibly bright" או "Blind you".
*   **מתי נקראת:** ידנית בתוך `onSensorChanged()`.

## 4. מחזור חיים
*   **`onCreate()`**: הגדרה ראשונית.
*   **`onResume()`**: רושמת את המאזין (Listener). זה חשוב: האפליקציה מתחילה "להקשיב" לחיישן רק כשהמשתמש מסתכל על המסך.
*   **`onPause()`**: מבטלת את רישום המאזין. **קריטי:** אם לא תעשו זאת, החיישן ימשיך לפעול וירוקן את הסוללה גם אם האפליקציה ברקע.

## 5. אינטראקציה עם הממשק (UI)
*   **אלמנטים:** `LinearLayout` (ID `LL`).
*   **קשר לקוד:** `findViewById(R.id.LL)`.
*   **אירועים:** מגיבה לשינויי אור פיזיים (אירועי חומרה).

## 6. לוגיקה כללית
1. המשתמש נכנס למסך.
2. האפליקציה מוצאת את חומרת חיישן האור.
3. כשהמשתמש מעביר את הטלפון מחדר חשוך לחלון בהיר, החיישן מפעיל את `onSensorChanged`.
4. הקוד בודק את ערך הלוּקס ובוחר צבע (למשל, צהוב עבור "Normal").
5. המסך הופך לצהוב באופן מיידי.

## 7. הסבר מפושט
**"הסבר במילים פשוטות"**
דמיינו שלטלפון שלכם יש **עין קטנה** (החיישן). מחלקה זו פועלת כ**מוח** שמפרש את מה שהעין רואה. אם העין רואה "כלום" (0 לוקס), המוח אומר למסך: "תהפוך לשחור!". אם העין רואה "אור שמש", המוח אומר: "תהפוך ללבן!".

---
**הערה לסטודנטים:** זכרו תמיד לבטל את רישום מאזיני החיישנים שלכם ב-`onPause()`. זהו כלל מס' 1 לאפליקציות אנדרואיד ידידותיות לסוללה.
